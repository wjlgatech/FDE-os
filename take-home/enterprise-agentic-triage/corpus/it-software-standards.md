# IT Software Standards

Effective date: 2025-03-01. Owner: Office of the CTO.

## Approved software list

Software and SaaS purchases must be for products on the approved-software
list in the IT service catalog. A product that is not on the approved list
requires an architecture review before purchase; requests for unlisted
products must be escalated to Enterprise Architecture.

## Security requirements

Any SaaS product that stores customer data must hold a current SOC 2
Type II report. A SaaS purchase without a current SOC 2 report on file
must not be approved automatically.

## Shadow IT

Purchases that attempt to acquire software under a non-software category
(for example, filing a SaaS subscription as "office supplies") violate
this standard and must be escalated.
