# Target JD — Google Cloud · Forward Deployed Engineer, Applied AI / GenAI

A **course-validation target** (Objective 1) and the **GCP-stack** anchor of the JD set (alongside
[Reflection](reflection-ai-fde.md), an AI-lab role, and [CVS](cvs-agentic-ai-engineer.md), an
enterprise role). It places the FDE nearest the **sales/go-to-market** end of the spectrum.

**Source:** captured from FDE-os research (`FDE-research-synthesis.md` + the ideation doc, which
fetched the live posting). Original posting: [Google Careers](https://www.google.com/about/careers/applications/jobs/results/81652014060053190-forward-deployed-engineer,-applied-ai,-google-cloud)
(Milan posting fetched 2026-06). _Re-verify against the live posting before relying on specifics._
**Org placement:** **Cloud Go-To-Market** — branded internally the "Agent Engineer." **Leveled:**
I / III / IV / Advanced. **Comp (Milan posting):** €116K–119K + 20% bonus + equity.

> Note the org-placement spectrum: the same FDE title spans from near-the-sales-quota (Anthropic,
> Google **Cloud**) to inside-the-research-lab (Google **DeepMind**, OpenAI). This target is the
> Google **Cloud** GTM role — *not* the DeepMind research-adjacent one.

## Responsibilities (verbatim signal)

- Build **production-grade agentic workflows** — e.g. **multi-agent systems** and **Model Context
  Protocol (MCP) servers** — for enterprise customers, in their environment.
- Stand up **evaluation pipelines and observability** for those agentic systems.
- **Convert friction points into reusable modules** — codify recurring solutions into shared assets
  (the "evolving playbooks" pattern, GTM-flavored).
- Partner with **sales and customer stakeholders** to scope and deliver real business outcomes
  (Cloud GTM placement → customer-facing from day one).

## Required / emphasized

- Strong **software engineering** background shipping **production** systems (**Python**, and the
  modern AI/agent stack).
- **Cloud / GCP** deployment — Vertex AI, GKE, and hybrid customer environments; DevOps practices.
- **Agentic system design**: agent orchestration, **multi-agent**, **MCP**, tool-use; common patterns
  named in the role's vocabulary include **ReAct**, **self-reflection**, and **hierarchical
  delegation**; frameworks such as **LangGraph**, **CrewAI**, and **Google ADK**.
- **Evaluation pipelines + observability** for LLM/agent systems.
- **Customer-facing** delivery with **high agency** in ambiguity.

## Competency clusters (the prep curriculum maps to these)

1. Production SWE — Python, production-grade
2. Enterprise / hybrid deployment + DevOps — **GCP** (Vertex AI, GKE), cloud, hybrid
3. Modern AI stack — evaluation pipelines, retrieval, observability
4. Agentic system design — **multi-agent**, **MCP servers**, orchestration (LangGraph / CrewAI / ADK; ReAct / self-reflection / hierarchical delegation)
5. The FDE craft — customer-facing GTM, scoping, converting friction into reusable modules
6. (light) Leadership — codifying reusable modules / playbooks; leveled I→Advanced

(enterprise), it triangulates the FDE competency demand across three very different employers.
