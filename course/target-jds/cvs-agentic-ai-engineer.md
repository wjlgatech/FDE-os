# Target JD — CVS Health · Agentic AI Engineer (Senior)

A **course-validation target** (Objective 1). Prep curriculum:
[`../prep/cvs-agentic-ai-engineer-prep.md`](../prep/cvs-agentic-ai-engineer-prep.md).

**Source:** provided by user (CVS Health posting, 2026-06-23). **Level:** Senior Software Engineer.

## Honest framing — FDE-adjacent, not a pure FDE role

This is an **Applied / Agentic AI Engineer** role: deep hands-on agentic-systems engineering for
enterprise, *"independently lead the delivery of scalable AI-powered products."* It is **not** a
forward-deployed/customer-embedded role in the Palantir-Delta sense — the spine is applied-AI
engineering depth, not on-site discovery. The overlap with FDE work is large (agentic systems, RAG,
evals, cloud/devops), so the technical clusters transfer directly. The **FDE craft** (the Delta
Loop, customer discovery) is a *differentiator to add on top* — it's what turns a strong Agentic AI
Engineer into someone who can also be deployed forward. The prep treats it that way.

## Core of the role (verbatim signal)

- *"Design and implement intelligent AI agents capable of reasoning, planning, tool usage, memory management, and multi-step workflow execution."*
- *"Build production-grade LLM applications using frameworks such as LangChain, LangGraph, CrewAI, AutoGen, Semantic Kernel."*
- *"Develop Retrieval-Augmented Generation (RAG) pipelines integrating vector databases and enterprise knowledge systems."*
- *"Develop prompt engineering strategies, evaluation pipelines, guardrails, and AI safety mechanisms."*
- *"Implement AI observability, monitoring, hallucination detection, and performance optimization."*
- *"Build scalable backend services and APIs using Python, Java, Node.js, or Go" · cloud-native on AWS/Azure/GCP · Docker/Kubernetes · CI/CD + IaC.*
- Leadership: *"Lead technical design discussions and architecture reviews · Mentor junior engineers · contribute to enterprise AI strategy."*

## Hard requirements

- **5+ years** software engineering; **2+ years** building GenAI/LLM applications.
- **1+ Claude MCP integrations** ← specific and non-negotiable; build a real one.
- **Python** + at least one of Java / Go / JavaScript-TypeScript.
- LLM APIs (OpenAI, **Anthropic**, open-source); vector DBs (**Pinecone, Weaviate, FAISS, ChromaDB**);
  RAG + semantic search; REST APIs, distributed systems; one cloud (AWS / Azure / GCP).
- Software design patterns, scalability, **observability**.
