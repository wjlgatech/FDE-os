# Staffing Confidence Scorecard
### Recommend people on evidence — not resumes. Deployable this week.

_A one-page leave-behind for an AI-delivery lead. No new platform: Microsoft Forms + Workday + one 200-line script._

---

**The problem.** One recommendation that looks great on paper and then no-shows, goes dark, or
can't deliver doesn't just cost one project — it burns the whole team's credibility to recommend
*anyone* again. Resumes miss it. "Will you be a reference?" misses it (the answer is always yes).

**The fix — three criteria, each measured by the instrument that actually predicts it:**

| Criterion | Instrument (this week, on Forms) | Hard rule |
|---|---|---|
| **Reliable** | Structured reference from someone who *worked with* them | one "went dark / no-show" ⇒ **automatic NO-GO** |
| **Technical** | 90-min **observed** work sample, one shared rubric | must clear the role's bar; *observed* = attributable |
| **Resourceful** | Learn an unfamiliar tool in a day + **teach it back** | must ship something that runs *and* teach it |

The three feed one **role-aware GO / NO-GO scorecard**. Reliability and resourcefulness gate every
role; technical gates technical roles. A strong score on one axis never rescues a weak one — a
reliable person who can't build is still a NO-GO for a build role.

> **Why it works:** 85 years of selection research (Schmidt & Hunter, 1998) — **work samples and
> _structured_ references are the top predictors of on-the-job performance; unstructured reference
> checks are near-noise.** The lever is *structure*, and this makes it repeatable.

**Roll it out in 5 days (zero procurement):**

1. **Mon** — paste 3 question sheets into Microsoft Forms; drop the kit in the team's Teams/SharePoint.
2. **Tue** — dry-run on 2–3 candidates already in the pipeline.
3. **Wed** — set one number: the work-sample pass bar per role (the only knob).
4. **Thu** — run a real staffing decision end-to-end; attach the scorecard to **Workday** + the **MyScheduling** note.
5. **Fri** — the whole center runs the identical funnel. Every recommendation now ships a defensible dossier, not a vibe.

**Fair & compliant by construction:** behavior-and-evidence only (no protected attributes),
identical questions for every candidate, anonymized ids, sanctioned tools only. It *advises* — the
recommender still signs.

**Scales to the real prize:** the same gate vets the *enablers* before they teach citizen
developers across a portfolio — using the exact learn-and-teach-back instrument that a tool
rollout already demands.
